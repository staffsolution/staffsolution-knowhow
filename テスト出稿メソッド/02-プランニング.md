# 広告プランニングの5ステップ

## 5つの設定項目

1. クリック単価の目安
2. キャンペーン設定
3. 出稿キーワード
4. 除外キーワード
5. 広告文

---

## ステップ1：クリック単価の目安

**基本方針：まず100円で出す**

- アルバイト・パート系は50円からでも可
- 100円で広告が表示されなければ翌日50円ずつ上げてテスト
- 表示されなければ広告費は発生しないため、低単価スタートのリスクは小さい

**キーワードプランナーが使える場合：**
- 「上部掲載に必要な目安単価」を参考に設定できる
- ただしアカウント開設後でないと使えない仕様

**応募率の目安：**

| キーワード種別 | 応募率目安 |
|---|---|
| 自社名求人（会社名で検索） | 3〜5% |
| 顕在層向け一般求人キーワード | 約1% |
| 比較検討段階のキーワード | 0.5〜1% |
| 関心が浅い層 | 0.1〜0.2% |

---

## ステップ2：キャンペーン設定

**地域設定はキャンペーン単位**のため、全国と地域を分けるなら別キャンペーンが必要。

**テスト出稿の標準設定：**

| 項目 | 設定値 |
|---|---|
| 日予算 | 1,000円（しっかりテストするなら5,000円も可） |
| 初期クリック単価上限 | 100円 |
| ディスプレイネットワーク | **必ずオフ** |
| 検索パートナー | **必ずオフ**（不正クリック対策） |
| 年齢ターゲティング | ターゲットに合わせて設定（シニア向けなら55歳以上） |
| 性別 | 必要に応じて設定 |

**重要：** 検索パートナーをオンにしたまま運用すると、IndeeJapanやTopSeekなど海外サイトからの不正クリックが発生するケースがある（確認した9アカウント全部で発生していた）。必ずオフに設定すること。

**「橋をかける」設計：**
```
ターゲット → 検索キーワード → 広告文 → LP → 応募
```
この流れのどこかが欠けると応募獲得は難しくなる。テスト出稿段階では「LPへの流入まで」を評価対象とすれば十分。

---

## ステップ3：出稿キーワード

**設定方法：**
- マインドマップで整理済みのキーワードをそのまま広告グループに貼り付けるだけ
- **マッチタイプはフレーズ一致で統一**（部分一致は使わない）

**キーワード戦略：**
- 一般的な求人キーワードだけでなく、**ターゲット起点のキーワード**（例：「シニア 住み込み」）を重視
- 競合が少なく採用単価を抑えやすいブルーオーシャンになり得る

**広告グループを分ける基準：**
- 広告文が異なる場合に分ける（テスト出稿段階では1グループでまとめても可）

---

## ステップ4：除外キーワード

テスト段階では厳密に作り込まなくてよいが、以下は最初から設定しておく：

| 除外カテゴリ | 例 |
|---|---|
| 対象外の雇用形態 | アルバイト、パート（正社員のみ募集の場合） |
| 対象外エリア | 配信地域以外の都道府県・都市名 |
| 対象外の業種・職種 | 採用対象外の職種名 |
| 同業者・顧客の検索語 | 集客目的のキーワード |

**重要：** 除外キーワードのマッチタイプは「部分一致のみ」で設定（通常と仕様が異なる）。

---

## ステップ5：広告文

**採用LP向けとnote向けで必ず分ける：**

| 遷移先 | 広告文の方向性 |
|---|---|
| 採用LP | 「求人スタッフ募集」など応募・採用を直接想起させる表現 |
| note記事 | 「失敗しない選び方」などお役立ち・情報提供系の表現（応募可能と誤認させる表現はNG） |

**文字数制限：**
- 広告見出し：1つあたり15文字以内（最大15個）
- 説明文：全角45文字以内

**作成手順：**
1. AIにプロンプトを貼り付けて広告文を生成
2. 文字数オーバーの箇所を手動で調整
3. 出稿対象のキーワードを見出しにまんべんなく含める（品質スコア改善のため）
4. テスト段階では日本語の自然さより**キーワード反映を優先**してよい
