# データ分析と改善

## 配信開始後に確認すること

### 1. 検索語句レポート

キャンペーン → 広告グループ → チェックを入れる → 検索語句

- 実際にどんなキーワードで表示・クリックされたかが確認できる
- 全検索語句の6〜7割程度しか表示されない（残りはブラックボックス）
- CSVでダウンロードして詳細分析も可能

**見るポイント：**
- 想定外のキーワードで流入していないか → 除外キーワードに追加
- 事前想定していなかった有望ターゲットが見つかっていないか → 新しい広告グループ候補に

---

### 求人広告のブラックボックス化について（HECポリシー）

**求人広告は特にブラックボックス率が高くなる傾向がある。**

GoogleのHEC（Housing, Employment, Credit：住宅・雇用・クレジット）ポリシーが適用される求人広告では、Googleのプライバシー保護強化の流れにより、検索語句が通常より見えにくくなるケースが増加している。

**実例：** 保育士領域（フレーズ一致）でのブラックボックス率が**75%**に達したケースあり。表示回数307回のうち検索語句が確認できたのは79クリック分のみ。

**領域別の傾向：**
- 介護・保育・医療・看護など社会的センシティビティの高い職種ほど顕著
- フレーズ一致・部分一致はさらにブラックボックス率が上がりやすい
- 完全一致はやや見えやすい

**代行・コンサル時の注意点：**
- クライアントのアカウントでもブラックボックス化が起きると、データ検証が難しくなる
- 「見えないから成果がない」ではなく、「見えているデータで判断する」という姿勢が重要
- 完全一致中心の設計にすることで、ある程度透明性を確保できる

**対処方針：**
- 検索語句が少なくても、クリック率・CVRなど見えている指標で判断する
- 完全一致キーワードの比率を上げてデータの可視性を高める
- ブラックボックス率が高い場合でも、「有効な流入があるか」はCV数で判断できる

### 2. 地域レポート

キャンペーン → 地域

- 都道府県・市区町村別の表示・クリック分布を確認
- 特定エリアへの絞り込みテストの参考になる

### 3. オーディエンス（年齢・性別）

- 想定と実際の反応が異なることがある（例：男性想定→実際は女性のクリックが多い）
- 有望なセグメントを発見したら、絞り込んで専用訴求のテストへ

### 4. デバイス

- 多くの場合スマホが9割以上
- LP制作はスマホ最優先で設計する

## 入札調整のサイクル

```
100円でスタート
→ 表示なし or インプレッションシェア10%未満
→ 50円ずつ上げる
→ インプレッションシェア50%を目安に調整
→ クリックと応募見込みを確認
```

**手動入札（個別クリック単価）に切り替えると：**
- 推定入札単価が表示されるようになる
- キーワードごとに単価を変えられる（例：注目キーワードは150円、それ以外は100円）

## テストログの記録

日付・上限CPC・マッチタイプ・対象キーワード・変更意図を記録しておく。

```
例：
4/6 - 上限100円でスタート（全キーワード）
4/7 - 「シニア 住み込み」を完全一致で追加、150円設定
4/8 - 個別クリック単価に変更
```

管理画面の「変更履歴」でも確認できるが、意図まではわからないのでメモを残す。

---

## 想定外キーワードを発見したときの対応

1. キーワードプランナーで検索ボリュームを確認
2. 有望と判断したら新しい広告グループを作成
3. そのキーワードに合った広告文とnote記事（または採用LP）を用意
4. テストを継続
