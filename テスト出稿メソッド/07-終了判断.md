# テスト出稿の終了判断基準

## 結論：十分なデータが集まったと判断できれば、いつ終わってもよい

1日でデータが集まったなら1日で終わってよい。

---

## 判断の考え方

1. **上限とする目標採用単価を先に決める**
2. そこから許容CPCと必要月間アクセス数を逆算する
3. 許容CPC内で必要アクセスが現実的に集まるかで継続/終了を判断

**計算例：**

```
目標採用単価：10万円
応募率：0.5%（悲観ケース）
採用転換率：20%

必要アクセス：1,000人/月
許容CPC：100円（10万円 ÷ 1,000アクセス）

→ 100円で月1,000アクセス集まりそうなら「いける」
→ 無理なら200円まで許容を広げて再評価
```

---

## 1日実績からの外挿判断

- 1日20アクセス集まっていれば → 月約600アクセスの見立て
- 目標が月500アクセスなら → 「いける」と判断して本実装へ進める

---

## 継続不可の基準

- 許容CPCをいくら上げても必要アクセスに到達しない
- → 検索ボリューム不足が根本原因
- → テスト出稿を終了し、方針転換を検討

---

## テスト出稿で達成不可と判断したときの選択肢

| 選択肢 | 内容 |
|---|---|
| キーワード拡張 | ターゲットを広げて別キーワードでテスト |
| 採用目標期間の見直し | 月1採用 → 3〜6ヶ月で1採用に変更 |
| Indeed出稿の追加 | リスティング広告以外のチャネルを追加 |
| ハローワーク活用 | 認知拡大 |
| 人材紹介会社との連携 | 候補者獲得 |
| 直接LP（ジョブリスティング以外）の設置 | ダイレクト流入導線の追加 |
