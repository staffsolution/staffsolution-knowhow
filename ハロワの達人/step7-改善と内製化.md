# Step7 改善・内製化

## 改善

### なぜ改善が必要か

- 最初から完璧な求人票は作れない
- 実際の答えを持っているのは求職者
- 改善を繰り返すことで精度が上がる（1回目より2回目、2回目より3回目）

ハローワークは無料で何度でも試せるので、数字で追いにくい制約はあるが、テストと改善を繰り返しやすい。

---

### 改善できる要素

| 要素 | 見直しの基準 | 戻るステップ |
|---|---|---|
| ターゲット | メインターゲットが全く応募してこない | Step1 |
| ベネフィット | 応募はあるが「思っていたのと違う」と断られる | Step3・4 |
| 求人票の書き方 | 魅力があるのに伝わっていない・抽象的 | Step6 |
| スカウト | 返信率が低い | Step6のスカウト部分 |

---

### 材料を集める

**面接で聞く：**
- 応募理由：「なぜこの求人に応募されたんですか？」
- 他に見ている求人：「他にどんな会社を見ていましたか？」
- 不安・疑問：「求人票を見て不安に感じたことはありますか？」

**辞退者に聞く（可能であれば）：**
「他に決められた理由を教えていただけますか？当社を辞退された理由も教えていただけますか？」

**入社したスタッフに聞く：**
- 「求人情報を初めて見た時どんなふうに思いましたか？」
- 「不安だったことはありますか？」
- 「自分に合っていると思ったのはどういうところでしたか？」

**ハローワーク職員に聞く：**
- 「どのあたりに関心を持たれたんでしょうか？」
- 「どんな点が刺さったと思われますか？」
- 「分かりにくい点はありましたか？」
- 「最近どんな条件で探している人が多いですか？」

---

### 問題を特定して改善する

| 症状 | 原因と対処 |
|---|---|
| 応募ゼロが続く | ターゲットの母数が少ない → ターゲットを見直す |
| メインターゲットが来ない | 求人票に刺さっていない → 書き方・ベネフィットを見直す |
| 競合に負けている（辞退が多い） | Step2・4に戻って競合・ポジショニングを見直す |
| 「思ったのと違う」と断られる | ベネフィットが抽象的・イメージとギャップあり → Step3・6を見直す |
| 魅力があるのに伝わらない | 具体性が足りない → Step6で求人票を書き直す |

---

### 改善のサイクル

1. 求人票を出す
2. 2〜4週間様子を見る
3. 声を集める（面接・ハローワーク職員・スタッフ）
4. 問題を特定して改善する
5. また出して様子を見る → 繰り返す

---

### 複数パターンをテストする

同じ会社・同じ職種でもターゲットとベネフィットの組み合わせを変えて複数パターン試せる。

**例（介護職の場合）：**

| パターン | ターゲット | ベネフィット | 主な訴求 |
|---|---|---|---|
| A | 資格あり・経験が浅い人 | マンツーマン研修で感覚を取り戻せる | 安心復帰 |
| B | 無資格・未経験・介護に興味がある | 働きながら資格が取れる | スキル習得 |
| C | 介護経験あり・無資格 | 経験を活かせる・資格取得支援あり | 即戦力＋成長 |

**注意：** ハローワークは同じ職種・同じ条件で複数パターンを同時出稿できない。1つずつ順番にテストする。

## 内製化

### 目的

- 社内でPDCAサイクルを回せるようにする
- 誰でも取り組める見える化
- 求人活動を俗人的なものから再現性のある仕組みに変える

---

### ハロワ求人管理シートの構成

**Googleドライブにフォルダを作成して管理する：**

```
採用フォルダ/
  ├── ハロワ求人管理シート（マインドマップなど）
  ├── リサーチ資料（Step1〜4の情報）
  ├── 求人票PDF
  ├── スカウト運用ドキュメント
  └── 反応記録（スカウト返信スクショ・応募者情報）
```

---

### 管理シートに記録する内容

**1. リサーチ情報**
- ターゲット設定〜ポジショニングのフォルダURL

**2. テストパターン（A・B・C・D）**
- ターゲット・ベネフィット・訴求ポイントをパターンごとに整理

**3. 運用ログ（1回のテストにつき1つ）**

| 項目 | 内容 |
|---|---|
| 選んだパターン | A・B・Cのどれを選んだか |
| 選んだ理由 | なぜそのパターンにしたか |
| 求人票 | PDFのURL・重点訴求ポイント・改善方針 |
| スカウト運用 | 検索条件・送った相手リスト・反応記録 |
| ハローワーク依頼記録 | 依頼日・求人番号・求職公開番号・依頼先HW・結果 |
| 面接ヒアリング | 応募前の悩み・応募の決め手・面接後の感想 |
| 改善点 | 何を変更するか・次のテストに向けた学び |

---

### AIを使った改善

```
応募者から「○○が不安」という声がありました。
この不安を解消するよう求人票を改善してください。
```

```
スカウト文の返信率が低かったです。
改善案を3つ提案してください。
```

---

### まとめ

1. 求人票を出す
2. 実際の声を集める
3. 一定期間取り組んで振り返る
4. 改善してまた出す

これをぐるぐる回すことで、どのパターンが反応を取れるか分かってくる。このサイクルを記録として残すことで、担当者が変わっても同じ水準で取り組めるようになる。
