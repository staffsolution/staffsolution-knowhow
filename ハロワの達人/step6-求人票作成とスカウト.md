# Step6 求人票作成・スカウト

## 求人票作成

### 準備

**リサーチ情報を声に出して読む：**
声に出して読むと情報の過不足が見えてくる。追記や修正をしてからAIに渡す。

**ハローワークインターネットサービスへのログイン確認：**
事業主マイページにログインできるか事前に確認する（ログインできないトラブルが多い）。

---

### 求人票の入力画面について

ハローワークインターネットサービス → 事業主の方 → 求人者マイページ → 新規求人情報を登録

入力は1〜8ページ構成。一時保存が可能だが**保存期間は2週間**。それを過ぎると入力内容が消えるので注意。

**公開設定の選択肢：**
- 求人情報を公開する → 誰でも閲覧可能・Indeedなど他媒体にも自動掲載・営業電話が来る
- ハローワーク求職者のみ → 登録者限定

**オンライン自主応募：**
- 受け付ける → 求職者がマイページから直接応募できる・メールでやり取りして面接調整まで可能
- 受け付けない → ハローワーク紹介状が必須

---

### AIフル活用で求人票を作る

**事前準備：AIにリサーチ情報を渡す**

```
これからハローワークの求人票を作成します。
まず以下のリサーチデータを把握してください。

【ターゲット】○○
【自社の特徴・ベネフィット・根拠】（Step3の内容をコピペ）
【ポジショニング】○○

これから作成するすべての項目で、ターゲットの不安を和らげ安心感を与える
温かい表現を使ってください。事務的な表現は避けてください。
準備完了なら「準備完了です」と返してください。
```

**第1段階：追加情報が不要な項目から作る（AIに指示する）**

- 職種名（28文字）：「職種＋ベネフィット」の形で5パターン
  - 例：「介護スタッフ／家庭両立できる日勤のみ」
- 仕事内容（360文字）
- 求人に関する特記事項（600文字）

**第2段階：追加情報が必要な項目**

- 就業時間に関する特記事項（120文字）：「ターゲットが生活をイメージできる温かい表現で」
- 休日（60文字）
- 福利厚生・研修制度・両立支援

---

### 注意点

- **AIが出した文章をそのまま使わない。** 必ず自分で読んで確認・修正する
- **文字数をチェックする。** Chromeの「簡単文字数カウント」拡張機能が便利
- **抽象的な言葉はスタッフの具体的な言葉に置き換える**（Step5で集めた言葉を使う）
- 足りない情報はAIに追加で依頼する

---

### 事業所情報も重要

事業所情報設定で写真を最大10枚登録できる。どんな人が働いているか・職場環境が分かる画像を入れることを強く推奨。

LPのQRコードを画像として入れることも可能。

## スカウト機能

### 概要

ハローワークに登録している求職者のプロフィールを検索して、直接リクエストを送れる仕組み。

- 希望職種・勤務地・資格・職歴などで検索できる
- マイページを持つ求職者へはオンラインで直接リクエスト送信
- マイページのない求職者へはハローワーク担当者経由で依頼
- 1求人あたり上限10件・8日間でリセット・同じ求職者への再送は不可

---

### 活用のコツ

- **検索条件は広めに設定する**（絞りすぎるとヒットしない）
- 求人票が魅力的でないとスカウトの反応も落ちる（求人票の質が前提）
- 返信があったらできるだけ即レスポンスする

---

### スカウト文をAIで効率的に作る：プロジェクト機能

ChatGPTのプロジェクト機能に求人情報を読み込ませておくと、毎回リサーチ情報を貼らなくてもスカウト文が作れる。

**プロジェクトの作り方：**
1. ChatGPTサイドバー → プロジェクト → 新規作成（名前：「ハローワークスカウト運用」など）
2. 「指示」にスカウト文の生成指示を入れる
3. 「ファイル」に求人票PDFをアップロード

**運用方法：**
求職者のプロフィールページをコピーしてプロジェクトのチャットに貼り付けるだけで、その人向けのスカウト文が自動生成される。

**スカウト文の仕様：** 件名30文字・本文300文字

AIが出した文章は必ずアレンジして送る。文字数が足りない場合は追記指示を出す。

---

### ハローワーク担当者経由でスカウトを依頼する場合

求職公開番号を控えてハローワークに電話し、「この番号の方にスカウトリクエストを送りたい」と依頼する（依頼先はその求職者が登録しているハローワーク）。

**LPがある場合：** QRコードを入れた送付状を作成し「ハローワーク担当者に一緒に渡してほしい」と頼む。OKをもらえれば、求職者がQRコードを読み込んでLPを見られる。

## 求人票AI指示文（詳細版・コピーして使う）

### 事前準備（AIにリサーチ情報を渡す）

```
これからハローワークの求人票を作成します。
まずは以下のリサーチデータを把握してください。

【募集する仕事】
・業種：（例：介護業界、運送業界）
・職種：（例：介護職、配送ドライバー）

【ターゲット設定（Step1）】
（マインドマップの内容をコピペ）

【自社リサーチ（Step3）】
（マインドマップの内容をコピペ）

【ポジショニング（Step4）】
（マインドマップの内容をコピペ）

【重要】
これから作成するすべての項目で、ターゲットの不安をやわらげ、
安心感を与える温かい表現を使ってください。
事務的な表現は避けてください。

以上を把握できたら「準備完了です」と返してください。
```

### 第1段階：追加情報なしで作れる項目

**①職種名（28文字）**
```
職種名を28文字で5パターン作ってください。
「職種＋ベネフィット」の形で。
例：「介護スタッフ/家庭と両立できる日勤のみ」
```

**②仕事内容（360文字）**
```
仕事内容を360文字で作ってください。
冒頭90文字は共感を生む言葉で、残りは具体的な業務内容で。
求職者が「自分にもできそう」と思える内容にしてください。
専門用語は避け、動作で書いてください（「支える」「組み立てる」など）。
```

**③事業所からのメッセージ（600文字）**
```
事業所からのメッセージを600文字で作ってください。
会社の想い、職場の雰囲気が伝わるように。
```

**④求人に関する特記事項（600文字）**
```
求人に関する特記事項を600文字で作ってください。
入社後のサポート体制や職場の安心材料を具体的に。
```

### 第2段階：追加情報が必要な項目

**⑤就業時間に関する特記事項（120文字）**
```
就業時間特記を120文字で作ってください。
【実際の情報】
・勤務時間：○○
・残業：○○
ターゲットが生活をイメージできる温かい表現で。
例：「午前9時〜13時の4時間勤務。お昼前には帰れるので家事や趣味の時間もしっかり確保できます」
```

**⑥休日等その他（60文字）**
```
休日等その他を60文字で作ってください。
【実際の情報】
・休日：○○
・年間休日：○日
ターゲットが働くリズムをイメージできるように。
```

**⑦事業内容（90文字）**
```
事業内容を90文字で作ってください。
【実際の情報】
・創業：○年
・地域：○○市
・事業：○○
地域性や温かみが伝わるように。
```

**⑧会社の特長（90文字）**
```
会社の特長を90文字で作ってください。
働く人目線で、安心感が伝わるように。
```

**⑨選考に関する特記事項（60文字）**
```
選考特記を60文字で作ってください。
【実際の情報】
・面接回数：○回
・面接場所：○○
ターゲットの不安をやわらげる温かい表現で。
例：「面接は和やかな雰囲気で。服装は普段着でOKです」
```

**⑩福利厚生（510文字）**
```
福利厚生を510文字で作ってください。
求職者が「この会社なら安心」と感じる内容に。
```

**⑪研修制度の内容（144文字）**
```
研修制度を144文字で作ってください。
未経験でも安心して学べることが伝わるように。
```

**⑫両立支援（510文字）**
```
両立支援を510文字で作ってください。
家庭や介護と両立しながら働けることが伝わるように。
```

## スカウト文AI指示文（ChatGPTプロジェクト設定用）

プロジェクトの「指示」に以下を入力する：

```
あなたは求人マーケティングと採用コピーの専門家です。
これからハローワークのリクエスト（スカウト）文を作成します。

【目的】
リクエスト文で応募意欲を上げること

【作成手順】
1. 求人票とユーザーが送る求職者情報を参照する
2. マッチングポイントを一言で示す
   （例：「あなたの○○な経験が、△△な職場で活かせます」）
3. リクエスト文を作成する
   ・件名：全角30文字以内
   ・本文：全角300文字以内
   ・文体：誠実で温かい
4. 改善案を2つ提案する

【重要】
・求人の説明より「歓迎の気持ち」を優先
・抽象語（「アットホーム」など）は避ける
・具体的な言葉で書く
```

プロジェクトの「ファイル」に求人票PDFを追加しておく。

**運用方法：** 求職者のプロフィールページをコピーしてチャットに貼り付けるだけでその人向けのスカウト文が生成される。AIの文章は必ずアレンジして送る。
