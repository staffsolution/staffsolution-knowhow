# Step3 自社リサーチ

[← 目次に戻る](./index.md)

---

## 目的

- 他社と比較された時に自社が選ばれる理由を見つける
- ターゲットの悩みや願望に応える形で自社の特徴を言語化する

**ゴール：** 特徴・ベネフィット・根拠の3点セットが揃っている状態

---

## 進め方（6ステップ）

### Step3-1：ターゲットを確認する

Step1で設定したペルソナを見返す。**「この人は何を求めているのか」を常に頭に置きながら進める。**

### Step3-2：どう比較されるか確認する

Step2の競合リサーチを見返す。ターゲットは何と比較して仕事を選ぶか。

### Step3-3：自社の特徴を集める

以下の枠組みで自社の情報を書き出す。

**働きやすさ：**
- 勤務時間：何時〜何時、残業はどのくらい、シフトの仕組み
- 休日：年間休日日数、有給取得状況、急な休みへの対応
- 人間関係：どんな人が働いているか、チームか個人か、雰囲気
- 職場環境：清潔さ、設備、働きやすい工夫

**待遇：**
- 給与・賞与・福利厚生・手当の内容

**やりがい：**
- 仕事内容の魅力、キャリアアップの道、評価制度、習得できるスキル・資格

困ったらAIに相談：
```
○○業界で○○をターゲットにしています。
働きやすさ・待遇・やりがいの視点で書き出してください。
```

---

### Step3-4：特徴をベネフィットに変換する

**特徴**（会社の事実）→ **ベネフィット**（求職者にとってどういいか）

特徴のままでは「どこも同じ」に見える。ベネフィットまで届けることで心に刺さる。

**「だから何？」を3回繰り返す：**

```
特徴：残業がない
 → だから毎日18時に帰れる
 → だから子どものお迎えに行ける・家族と夕食が取れる
 → だから家庭を大事にできて安心して働ける

特徴：1ヶ月マンツーマン研修がある
 → だから先輩がずっと教えてくれる
 → だからブランクがあっても一つずつ覚えられて質問しやすい
 → だから失敗を恐れずに働けて安心して続けられる
```

AIに相談：
```
以下の特徴をターゲット「○○」に向けてベネフィットに変換してください。
【特徴】○○
```

---

### Step3-5：ベネフィットを裏付ける根拠を出す

「それって本当なの？」に答える具体的な証拠。

| ベネフィット | 根拠の例 |
|---|---|
| ブランクがあっても安心して始められる | 50ページの研修手順書あり・最初の2週間は軽作業のみ・夜勤は半年後から |
| 急な休みが取れる | シフト共有システム導入・前日申請でも基本承認・急な休み対応月3件の実績 |
| 家庭を大事にできる | 40代主婦が8割・有給取得率92% |

数字・ルール・体制・実績で裏付けると説得力が増す。

AIに相談（根拠が思いつかない場合）：
```
「○○というベネフィット」を伝えたい。
どんな根拠があれば信頼できるか教えてください。
```

---

### Step3-6：悩み・願望ごとに「選ばれるポイント」を書く

ターゲットの悩み1つずつに対して、特徴・ベネフィット・根拠をセットで書く。

**例：子どもの急な体調不良で休めるか不安（主婦のターゲット）**

| 項目 | 内容 |
|---|---|
| 比較先 | スーパーレジ・学校給食・清掃パート |
| 特徴 | 急な休みに対応できる |
| ベネフィット | 前日でも休める → 罪悪感なく休める → 家庭を優先しながら安心して働ける |
| 根拠 | シフト共有システムで前日申請でも基本承認・スタッフ40代主婦8割・休暇取得率92% |

これを主要な悩みごとに繰り返す。

## 大事なこと

- **求職者目線**で考える（会社にとっていいことではなく求職者にとっていいこと）
- 「丁寧に指導します」「アットホーム」などの抽象語は使わない
- **具体的な数字・事実・エピソード**で表現する
- 特徴・ベネフィット・根拠は**必ずセット**で書く
