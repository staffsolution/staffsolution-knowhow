# 仮説思考 ノウハウ

9ステップ実践ガイドから抽出した、リサーチ・制作・改善の全ステップで使う思考フレーム。

最終更新：2026年4月9日（競合LPのない分野の戦略・採用難度確認を追記）

---

## 核心原則 {#core}

**自走できる人とそうでない人の差の多くは、「仮説思考を持っているか」にある。**

仮説思考とは「多分こうすればうまくいく」という**見立てを持って動くこと**。  
見立てがないと観察・検証の軸がなく、AIへの質問も漠然とし、フィードバックも消化できない。

仮説思考はセンスではなく**型で身につく**。目的→観察→思考→実行→検証を繰り返すことで精度が上がる。

---

## 仮説思考5ステップの全体像 {#overview}

```
① 目的    ：何のためにやるのか、狙いを定める
② 観察    ：現状を正確に見る・整理する
③ 思考    ：「もし〇〇なら」で仮説を立て、情報を追加する
④ 実行    ：仮説を分解して一つずつ確認する
⑤ 検証    ：結果を目的に照らして振り返り、観察・思考へ戻る
```

**このサイクルは一度で完結しない前提で回す。**  
AIはこの5ステップの速度と精度を上げる相棒。選択と判断は人間が行う。

## ステップ1：目的 {#step1}

### なぜ「目的」から始めるのか

目的が曖昧だと、以降の作業の大半が無駄になりやすい。  
LP制作・リサーチ・広告出稿はすべて**手段**。目的ではない。

### 目的の具体化

```
「それは何のために？」を繰り返して深掘りする

例：
「LPを作る」→ 何のため？
→ 「応募を増やす」→ 誰の・どの課題を解くため？
→ 「未経験の主婦が安心して一歩踏み出せるように」→ 数値でいくつ？いつまでに？
```

**目的の3点セット：**
1. 誰の・どの課題の・どんな変化のためか
2. 成功基準（数値・件数・期間）
3. どこまでやれば完成か（完成形のイメージ）

**手段を選ぶ前に採用難度を確認する：**

ジョブリスティング（リスティング広告＋LP制作）は、採用が難しい案件に使う手段。始める前に「本当にこの手段が必要か」を確認する。

```
まず無料媒体・紹介で試す：
→ パート・アルバイト・人気業態（カフェ・保育・医療事務など）は無料媒体で集まる可能性が高い

ジョブリスティングが必要なケース：
→ 正社員・専門職で採用が難しい
→ 多店舗展開していて大量に採用したい
→ 無料媒体を試しても全く集まらない状態が続いている
```

「念のためジョブリスティングをやろう」ではなく、「この案件はリスティングが必要か」を最初に問う。かける労力と得られる成果が釣り合うかを先に確認する。

**NGパターン：**
- 目的を決めずキーワードや競合だけ先に見る
- LP完成を最終ゴールにする
- 数値なしの曖昧な「応募を増やす」だけで止まる
- 採用難度を確認せずジョブリスティングを始める

## ステップ2：観察 {#step2}

### 観察の本質

観察とは「現状の正確な把握」。思い込み・バイアスを横に置き、事実とデータで現状を捉えること。  
整理なくして仮説の質は上がらない。

### 情報整理の手順（みんながつまずくポイント）

収集した情報をそのまま抱えても仮説は立てられない。以下の順で整理する：

```
1. 削除      ：ターゲットに不要な情報を捨てる
2. グループ化：共通点・パターンでまとめる
3. 並べ替え  ：流れ・重要度・時系列で順番をつける
4. 一言要約  ：各グループを「一言で何か」に凝縮する
```

AIに「このターゲットに必要な情報だけ整理して」と指示して補助させてよい。ただし最終確認は人間が行う。

### バイアスを防ぐ自問

- 「本当にそうか？」（経験則を疑う）
- 「事実・データで裏付けられるか？」
- 「別の解釈はないか？」

**設定した数値・母数を疑う習慣：**

ターゲットの経験年数や在職期間など「このくらいの人が多いはず」と設定したら、その数値を自分で疑ってAIに確認する。「本当にそのパターンが最も多いのか」を問わないまま進めると、母数が実態よりずれたターゲット設定になる。

```
疑問を持つ → 仮説を立てる → AIに投げる → 確認する
```

疑問を持てることは正しい。そのままにせず必ず確認まで進める。

**NGパターン：**
- AI出力をコピペだけで確定する（精読していない）
- 思い込みを疑わない
- 整理に時間をかけず仮説だけ立てる
- 疑問が浮かんだのにAIに投げず、思い込みのまま設計を続ける

## ステップ3：思考 {#step3}

### 観察と思考は往復で精度が上がる

観察が一段落したら仮説を立て、足りなければ観察に戻る。一度で完結しない。

### 仮説を深掘りする質問の型

```
なぜ？    ：なぜそう言えるのか、根拠を問う
何？      ：他に何があるか、別の要素を探す
他には？  ：視野を広げる
逆の場合は？：反例・例外を確認する
```

**視点を増やす問いかけ：**
- 相手（求職者）の立場から見るとどうか
- 競合と比べてどう見えるか
- 具体的な数字・シーン・事例に落とすと何か
- 時系列で見るとどう変化するか
- 逆説的に見るとどうか（「なぜ選ばれないか」）

**出稿まではすべて仮説。決めつけない。**

小学4〜5年生でも伝わる平易さを理想にする。難しい表現は読まれない。

**NGパターン：**
- 型なしで思いつきだけ並べる
- 一度決めた仮説を検証せず進める
- 難しい文章のまま出す

---

## ステップ4：実行 {#step4}

### 仮説を「大から小」へ分解して確認する

```
① 全体方向性を確認（ターゲット・路線にズレがないか）
② カテゴリ単位で確認（十分か・抜けはないか）
③ 個別項目を深掘り（理由→結果→ベネフィットの連鎖）
```

方向ズレは後工程のすべてを無駄にする。細部に入る前に全体方向を確認する。

### AIとの壁打ち

- AIは論点の洗い出しに強いが、選別・判断は人間が行う
- AIに聞く前に「自分の仮説と確認したい論点」を1行で書いてから質問する
- AI出力をそのまま最終答えにしない

### フィードバックの受け方

| よくある反応 | 正しい受け方 |
|---|---|
| 感情的になる・反論する | 感情を切り離し、情報として受け取る |
| 曖昧なまま次に進む | 「どこが・なぜ・他と比べて」と聞き返して具体化する |
| 言い訳する | 改善点を1つ以上言語化してから次の一手を出す |

**フィードバック依頼のとき：「何を見てほしいか」を明示する。「どう思いますか」だけでは質の高い指摘が返らない。**

---

## ステップ5：検証 {#step5}

### 検証は「量」ではなく「目的適合」で見る

情報が多く集まったことに満足しない。「当初の目的に対して、これは答えになっているか」で評価する。

### 検証の手順

```
1. 何を知りたかったかに立ち返り、不要な情報を捨てる
2. 数字・データ・事例で事実ベースにする
3. 仮説と結果のズレを見つけ、原因まで言語化する
4. 他の可能性（多角的チェック）で固定化を防ぐ
5. 気づきを記録してチームで共有する
```

**ズレは成長の材料。曖昧にしない。**  
「想定と違った」で終わらせず、「なぜズレたか」まで言語化することで次の仮説精度が上がる。

**NGパターン：**
- たくさん出たで満足して終わる
- 主観のみで「良さそう」と判断する
- ズレを言語化せず次に進む
- 気づきを個人の中に閉じて共有しない

---

## チームで仮説を共有すると成長が加速する {#team}

個人の気づきを組織資産にする仕組みを作る。

### 共有シートの記録項目

```
日付 / ステップ / 作業内容 / 立てた仮説 / 結果 /
フィードバック後の変化 / 次アクション / 自己評価
```

**運用のコツ：**
- フィードバック後にその場で1〜2分で簡潔に入力（完璧な記録より続くこと）
- 週次でチームに共有・ディスカッション
- AI会話のログを丸ごと貼らない（要点だけ転記する）
- 記録の目的は「分析」ではなく「次の行動」

## つまずき早見表 {#trouble}

| つまずきのサイン | まず見直すステップ |
|---|---|
| 何を書けばいいか分からない | ①目的（何のためか） |
| 訴求がブレる・刺さらない | ②観察（ターゲット・競合・ニーズ） |
| 根拠が弱い・説得力がない | ②観察＋③思考（比較・理由・具体化） |
| 施策が散らかる | ①目的（優先順位） |
| 改善しても数字が動かない | ⑤検証（測定・サンプル・仮説の当たり外れ） |
| チームで認識がバラバラ | 共有（前提・定義・テンプレ） |

※正解ではなく切り分けの起点。まず1行だけ当てはまりを決め、いきなり表現修正や全やり直しはしない。

## 自社リサーチへの適用 {#apply}

仮説思考5ステップは**自社リサーチで特にフル活用できる**。

| 仮説思考ステップ | 自社リサーチでの具体的な使い方 |
|---|---|
| ①目的 | 「誰のどんな応募を増やすか」を先に定義する |
| ②観察 | 収集した会社情報を「削除→グループ化→並べ替え→要約」で整理する |
| ③思考 | 「なぜ・何・他には・逆は」でベネフィットを深掘りし、ビッグプロミスの糸口を探す |
| ④実行 | 全体方向性（ターゲット・ポジション）を確認してから個別項目を詰める |
| ⑤検証 | スタッフインタビューの結果と仮説のズレを言語化し、次の仮説に反映する |

情報整理（②観察）でつまずく人が多い。「情報は集まったが何を訴求すべきかわからない」という状態は、整理の手順（削除→グループ化→並べ替え→要約）を飛ばしていることが多い。

## 競合LPがない分野はLPを作るだけで有利 {#no-competition-lp}

業界や分野によっては、リスティング広告＋LPで集客しているプレイヤーがほぼいない状態がある。不動産ポータルへの掲載が主流のテナント・貸しスペース業界がその典型例。

このような分野では：
- 質の高いLP1枚を作るだけで、競合がいない状態で問い合わせを集められる
- 「誰もやっていない」こと自体がポジショニングになる
- 無理にターゲットを絞らなくても、出すだけで成果が出やすい

仮説として「この業界でLPを作っている競合はいるか」を最初に確認する習慣をつける。競合がいなければ、クオリティより「出稿すること」を優先する判断が合理的になる。

```
競合LPがない分野 → 出す → 反応を見る → 改善する
競合LPが多い分野 → ポジショニング・差別化を先に固める → 出す
```

## beliefが強くて検索すら起きない層の設計 {#no-search-layer}

応募前停止期よりさらに手前に存在するターゲット層がいる。「そもそも仕事を探すという行動自体が起きていない」という層だ。

```
関心を持っていない
（働けると思っていないので、検索すら起きない）
　↓
偶然情報に触れる（チラシ・紹介・SNS）
　↓
「え、これ自分でもできるの？」という気づき
　↓
関心を持つ → 求人を見る → 応募前停止 → 応募
```

この層に対しては、リスティング広告（能動的検索に依存する手法）は機能しない。「ターゲットがいる場所に情報を届ける」設計が必要になる。

**beliefが強いターゲットの特徴：**

- 「どうせ自分には無理」という思い込みが先にある
- 情報を探しに行くという行動自体を起こさない
- 信頼できる人（知人・同じ立場の人）からの紹介で初めてアンテナが立つ
- 紹介ネットワーク（特にコミュニティが強い属性：ママ同士・同業者など）が機能しやすい

**設計のポイント：**

- リスティング広告より、チラシ配布・紹介・コミュニティへのアプローチが合う
- LPのリードは「え、これ自分でもできるの？」という驚きを引き出す一言を冒頭に置く
- belief（「保育園に預けられないと働けない」など）を冒頭で言語化し、その思い込みを壊すことからページを始める

## 応募前停止期を意識した仮説設計 {#pre-apply}

多くの求人が見落としているのが「応募前停止フェーズ」の存在。求職者はLP・求人票を見て「応募しよう」と思う前に、一度足が止まる段階がある。

```
関心を持つ → 求人を見る → 応募前停止 → 応募
                               ↑
                         ここで止まっている人が最も多い
```

「応募が来ない」のは多くの場合、求人票の訴求力の問題ではなく、この応募前停止期の不安を解消できていないことが原因。

**応募前停止期に求職者が抱えている不安の例：**

- 「未経験・年齢で落とされそう」（不採用への恐れ）
- 「ちゃんと教えてもらえるのか不安」（職場環境への不信）
- 「体力・人間関係についていけるか不安」
- 「履歴書を書くのが面倒・怖い」

**設計のポイント：**

応募前停止期の不安を先回りして言語化し、「うちはこう対応している」という根拠を見せることが、他社との差になる。

競合のほとんどはこのフェーズを無視して訴求だけ並べている。この層の不安に届けられれば、それだけで選ばれやすくなる。

## AIとの正しい組み合わせ方 {#ai-usage}

| 用途 | 人間 | AI |
|---|---|---|
| 仮説を立てる | ◎（見立てを持つ） | ○（たたき台を出す） |
| 情報を広く出す | △ | ◎（速い） |
| 選別・判断する | ◎ | ✗（最終判断させない） |
| 深掘り質問に答える | ◎ | ○ |
| 記録・整理を補助する | ○（確認する） | ◎（整理補助） |

**AIに「いい案ください」とだけ聞くのはNG。** 自分の仮説と確認したい論点を1行書いてから質問すると、出力の質が大きく変わる。
