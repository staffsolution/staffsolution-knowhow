## 核心原則 {#core}

**コピーライティングは「書く作業」ではなく「素材をレシピに当てはめる組み立て作業」。**

- 素材（リサーチ）がなければ書けない。白紙の状態で書こうとするから難しいと感じる
- レシピ（型・フォーマット）がなければ効果が出ない
- テクニックはスパイス。素材とレシピが揃っていれば誰でもそこそこのものが作れる

**「どう言うか」よりも「何を言うか」の方が重要**（デービット・オグルビー）。

## 進め方 {#steps}

### ステップ1：ヘッドラインを考える {#step1}

**ヘッドラインで反応の8割が決まる。**  
ファーストビューの大見出し（一番最初に目に入る情報）で注意を引けなければ、下を読んでもらえない。

**ヘッドラインの役割：** 求職者の悩み・願望と、会社の独自の強みを「つなげること」。

**ヘッドラインの3部構成：**
- プリヘッド：一番上に小さく表示。証拠の情報などを入れる
- ヘッドライン：一番目立つ見出し
- サブヘッド：ヘッドラインの補足・詳細

**入れるアイデアの候補：**
- ビッグプロミス（ターゲットが得られる最大の結果）
- 独自メソッド（ビッグプロミスを実現する会社独自の仕組み）
- ビリーフへの切り込み（「○○なんて無理」という思い込みへの反論）

**ヘッドラインのNGパターン：**
- 英語表記・抽象的な表現（「NEXT STAGE」「挑戦する価値がある」など）
- 他社のマネ・ターゲットが不明確（誰にでも当てはまる表現）
- ベネフィットの深掘りが途中で止まっている（「なんかいいそう」で終わる）
- 業界の専門用語（未経験者は意味がわからない）
- ウサン臭い（強すぎる約束に証拠がない）

**ヘッドラインのチェックポイント：**
- 一瞬で理解できるか（考えさせてはいけない）
- 読む理由を感じてもらえるか（飛躍していないか）
- 独自メソッドが入っているか（他社に真似されないか）

### ステップ2：ストーリーボードを作る（全体構成） {#step2}

映画の絵コンテや本の目次のようなもの。先に作ることで話の流れがブレない。

**ストーリーボードを作らないと起きること：**
- 途中で話がずれていく
- カテゴリーに違う情報が混ざる
- 論理的な矛盾が生じる

**型：4つのP（元世界ナンバーワンコピーライター ゲイリー・ベンシベンカの型）**

| P | 内容 | ページ内での役割 |
|---|---|---|
| **Problem（問題・願望）** | 求職者が抱えている悩みや願望を提示して共感させる | ヘッドライン＋オープニング |
| **Promise（約束・解決策）** | うちに入ればこうなれると伝える | ボディ前半（ベネフィット） |
| **Proof（証拠）** | 理由・データ・スタッフインタビュー動画で信用させる | ボディ中盤（証拠提示） |
| **Proposition（提案）** | 具体的な働き方・応募方法を伝えて行動させる | クロージング |

**条件を先に見せない：** 応募要項（給与・休日）はPromise・Proofの後。先に見せると条件だけで判断されて他社に流れる。

### ステップ3：ロジカルチェーンを組む {#step3}

A→B→C→D→E と因果関係でつながる論理の鎖を作ること。

**鎖を強くする方法：各情報を事実・証拠でバックアップする**
- 理由・根拠を必ず伝える（結論だけ言わない）
- 比較情報を入れる（他業種・他社・昔の自社との比較）
- 不安感を先回りして払拭する（求職者が持ちそうな疑念を先に解消する）

**論理的矛盾が起きるパターン：**
- 話が違う方向に飛ぶ（シニア向けに書いているのに「若者にもおすすめ」が入るなど）
- 論理が飛躍する（理由なしにいきなり結論）
- ターゲットの意識レベルを無視する

**【応用】悪化の流れを段階的に見せる（問題提起型ロジカルチェーン）**

問題点を列挙するだけでなく、「AだからB、BだからC、だから最終的にDになる」という悪化の連鎖を矢印で見せると、読み手が「ああ、あるある」と共感しやすくなる。その後に「当社はこうしている」という解決を提示すると、コントラストが際立つ。

```
（例：研修が不十分な他社のケース）

研修時間が少ない・専任トレーナーがいない
　↓
理解不足のまま現場に立つ
　↓
何を優先すべきかわからない・一つ一つの動きに自信が持てない
　↓
探り探りで動く中でミスが増える
　↓
先輩や社員から叱られる（何日も続く）
　↓
自信を失い「自分には向かない」と感じる
　↓
退職
```

この流れをまとめに使った後、自社の対比（「当社では：研修45時間・1対1専任トレーナー・マニュアル完備」）を並べるとコントラストが最大化する。

### ステップ4：ライティング（素材の当てはめ） {#step4}

ストーリーボードが完成したら、集めた素材を各パーツに当てはめる。ほぼ「プラモデルの組み立て作業」。

**書く際に意識する5つのポイント：**

**① 求職者の意識レベルに合わせる**

| レベル | 状態 | 対策 |
|---|---|---|
| レベル2 | 問題・願望はあるが解決策（業種）がわからない | **未経験者募集の主戦場。まず業種のベネフィットから伝えてレベル3へ引き上げる** |
| レベル3 | 業種（タクシー・新聞販売等）は決まったが会社が未定 | 業種のベネフィット→会社のベネフィットの順で伝える |
| レベル4 | 会社の存在を知っていて比較検討中 | 会社のベネフィット・差別化点を中心に伝える |

未経験者募集はレベル2の攻略が最大のポイント。レベル2の人に「タクシードライバー募集」と直接言っても反応しない。まず業種への誘導から始める。

**レベル2ターゲットへの誘導設計：**

「パート 子供関連」など広いキーワードで調べている人を、特定の業種（例：障害児支援）に誘導するケースでは、LPを以下の構成にする。

```
① 入口は広くとる（子供が好き・短時間から働きたい、など多くの人が当てはまる訴求）
② 読み進める中で差別化ポイントを伝える（保育園との違いなど他の選択肢との比較）
③ 「一人ひとりとじっくり向き合いたい人には、この業種が合っている」と絞り込む
④ 絞り込みに共感した人が応募に向かう
```

ターゲット設定の時点で業種を前面に出しすぎると母数が狭くなる。最初の入口をざっくりとした表現にして、読み進めるうちに「自分に向いている」と感じてもらう設計が有効。

**② 具体的に書く（抽象的表現を排除する）**

- 抽象的：「定着率が高い」「働きやすい」「仕事が楽しい」
- 具体的：「1年以上続けた社員の定着率90%以上」「月24日・8時間勤務が基本」「仕事終わりに社員と居酒屋で一杯飲む時間が好きです（スタッフの言葉）」

具体的な情報＝見れる・聞ける・触れる情報（シーン・数字・名詞）。

**③ イメージさせる**

- シーンを描く（エピソードをそのまま使う）
- テキストだけでなく写真・動画・グラフを活用

**④ デメリットを正直に伝える**

「オネスティ・イズ・ザ・ベスト・ポリシー」。デメリットを伝えることでベネフィットの信用性が上がる。  
ターゲットを絞り込んで「この人には絶対うちが最適」と心から思えてから書くと、デメリットを伝えながらも説得力のある文になる。

**⑤ フューチャーペーシング（入社後の未来を見せる）**

未経験者の「入社したらどうなるか」という不安を先回りして払拭する。
- 研修内容（期間・内容・担当者・給与）
- 1日の流れ（時間・写真・グラフ）
- 寮（広さ・距離・写真）
- 給与モデル（具体的な金額・時期）

### ステップ5：編集 {#step5}

**基本は「削ること」**（リサーチが足りない場合は付け足し）。

**チェック方法：**
1. ヘッドライン・見出しだけを抽出してチェック（本文は流し読みされるため）
2. リードの部分を重点的に確認（コンセプトがブレていないか・差別化できているか）
3. ボディ：入社後の変化・結果がイメージできるか
4. ロジカルチェーンが途切れていないか

**セルフチェックの方法：**
- **音読する**（つながりのおかしさ・飛躍が見つかる）
- **時間を置いてから読む**（気づかなかった問題点が見つかる）
- **他人にレビューしてもらう**（最重要。自分では気づけない部分がある）

## 補足・深掘り {#supplement}

### 繰り返し出るNGパターン {#ng}

- リサーチ不足の状態でライティングに入る（素材なしで書こうとする）
- ストーリーボードを作らずに書き始める（結果ぐちゃぐちゃになる）
- 抽象的な表現で止まる（「働きやすい」「成長できる」「アットホーム」）
- 意識レベルを無視して会社の説明から始める
- 募集要項を最初に見せる（条件合戦に巻き込まれる）
- デメリットを全て隠す（ミスマッチが増え・定着率が下がる）
- 音読・他人へのレビューをしない（編集の手を抜く）

### 冒頭での思い込み先出しパターン {#misconception}

読み手が「自分には関係ない」と感じて離脱する前に、その思い込みを冒頭で出して崩す構成。  
「とはいえ」構成（後述）が「強みを伝えた後の懸念処理」であるのに対し、こちらは**応募するかどうかの判断より前にある根本的な誤解を取り除く**ために使う。

```
① 業界・職種に対する思い込みを先出しする
   「美容室で働くには美容師の資格が必要、と思われるかもしれませんが、」
     ↓
② 「当店は違います」と伝える
   「当店の場合は問題ありません。なぜなら、」
     ↓
③ 理由（仕組み）を説明する
   「美容施術は美容師が、リラクゼーション施術はセラピストが担当する完全分業制だからです。」
```

**ポイント：「なぜなら」の中身を正しく選ぶ。**  
「専門技術は資格者が担当します」のように「それはそうでしょう」と読み手が感じる内容では理由として機能しない。  
「だから未経験でも大丈夫」と直結する仕組みや事実を入れる。

**使いどころ：**
- 業界に対する参入障壁の誤解（資格・経験・体力など）がある職種
- ターゲットが「どうせ自分には無理」と感じて読むのをやめてしまいそうな場合
- 冒頭（ヘッドライン直後のオープニング）に入れると効果的

### 「とはいえ」構成（懸念先取りパターン） {#tohaie}

強みを伝えた直後に、読み手が抱く「でも、どうせ…」という懸念を先取りして処理する構成。  
良い条件が並ぶほど「何か裏があるのでは」という疑念が生まれやすい。それを無視して強みだけ並べても信じてもらえない。

```
① 強みの提示
   「定年制度を廃止。60歳以降も手取り月20万円以上を目指せます。」
     ↓
② 読み手の頭の中（懸念が生まれるタイミング）
   「でも、そんな条件ならどうせ急な呼び出しや連勤があるんじゃないの？」
     ↓
③ 業界の実態を認める（共感・信頼を得る）
   「実際、多くの警備会社ではそういった環境が今も続いています。」
     ↓
④ 自社の転換を伝える（なぜ変わったのか＋結果）
   「当社は無理な受注をやめた結果、急な呼び出しも連勤もなくなりました。」
```

**ポイント：業界の実態をいったん認めること。** 否定せず共感してから自社の差を見せると、読み手の抵抗が下がる。

**NGパターン：**
- 強みを並べるだけで懸念への回答がない
- 「うちは違います」だけで言い切る（なぜ違うのかの因果関係がない）
- 業界背景の説明なしに自社の特徴だけ伝える（比較対象がないと価値が伝わらない）

### ターゲットの欲求を時系列ストーリーで設計する {#target-story}

ターゲットの欲求・悩み・願望は「今どういう状態か」を箇条書きで並べるより、時系列の流れで書いた方がLPの訴求に直結する。

**時系列ストーリーの骨格：**

```
① なぜ今なのか（きっかけ・生活の変化）
　  例：子育てがひと段落して時間に余裕が出てきた
　↓
② なぜ趣味・習い事ではなく仕事なのか（動機の分岐点）
　  例：子供が大学生になるとお金がかかるので家計の足しにしたい
　↓
③ どんな仕事でもいいわけではない（欲求のコア）
　  例：人の役に立てた実感が残る仕事がしたい
　↓
④ なぜこの業種なのか（業種選択の理由）
　  例：子供が好きで、一人ひとりと深く関われる環境で働きたい
　↓
⑤ とはいえこういった制約がある（現実的な条件）
　  例：家庭を最優先にしながら、週2〜3日・短時間から働きたい
```

この流れで書くと、読み手が「そうそう、自分もそう感じていた」と共感しやすくなる。また、この流れ自体がLP内の訴求順序のたたき台になる。

**よくある失敗：**

- 「現在の状況」と「欲求・要求」を別セクションに分けて書き、ストーリーの流れが途切れる
- ポジティブな動機だけ書いて、「なぜ今なのか」というネガティブな起点（焦り・問いかけ）を入れない
- 欲求を多く並べすぎて優先順位が見えなくなる（3点に絞るのが目安）
- 制約条件（勤務日・時間）を欲求と同列に書いて、何が一番大事かが伝わらない

### 「〇〇OK」への疑念を先回りする：証拠先出しパターン {#ok-claim}

「ブランクOK」「未経験OK」「子育て中OK」などの主張は、競合も同じ言葉を使っているため信頼されにくい。「うちはOKです」と書くだけでは「本当に？」という疑念が先に来る。

**なぜ疑念が生まれるか：**

業界あるあるとして「OKと書いているが実態はそうでもない」という経験を読み手が持っていることが多い。特にブランク復帰・子育て中など、一度は諦めた経験のある層ほど疑念が強い。

**解消の設計：証拠を先に出してから理由を続ける**

```
① 証拠情報を先に出す
　  例：「ブランク2〜10年のスタッフが○人在籍。定着率○%。」
　↓
② その主張が本物である理由（仕組み・サポート体制）を続ける
　  例：「社長との個別面談で自分のペースに合わせた復帰プランを一緒に作れるから。」
　↓
③ よくあるシーン・事例で「自分のことだ」と感じさせる
　  例：「2歳の子供を週2日だけ預けながら、月2〜3日・午前だけ働いている方もいます。」
```

「OK」という主張が大きいほど、証拠も大きくしなければならない（逆説：約束が大きいほど証拠が必要）。

**適用場面：**
- ブランクOK・未経験OK・子育て中OKなどの訴求
- 「本当に大丈夫なのか」という疑念が読み手に生まれやすい主張全般
- 業界の慣習から「それは名ばかりでは」と思われやすい条件

### 業界のネガティブイメージを逆転させる訴求の型 {#image-reversal}

業界全体にネガティブなイメージが定着している場合、「冒頭での思い込み先出し」よりもさらに一歩踏み込んで、**「業界のイメージ一覧 → うちは真逆」という比較テーブルで見せる**と一目で差が伝わる。

求人LPで特に有効な業種：飲食・介護・警備・建設など「汚い・きつい・ブラック」のイメージが先行しやすい業種。

**設計の流れ：**

```
① 業界に対する世間のネガティブイメージを先に言語化する
　  例（ラーメン店）：「油ぎって汚い」「スタッフの服や髪に匂いがつく」「ブラックな働き方」

② 「当社は真逆です」と伝える

③ 各イメージに対応する「当社の実態」を証拠とともに出す

比較テーブルの例：
| 業界のイメージ | 当社の実態 |
|---|---|
| 油ぎって汚い厨房 | タイル・ガラス・ステンレス素材で清潔を維持 |
| スタッフに匂いがつく | 月1回クリーニング業者が清掃。匂いが残りにくい |
| ブラックな働き方 | シフト提出制・子供の急な発熱でも当日対応可 |
```

**ポイント：**
- 「うちは違います」だけで終わらず、「なぜ違うのか（仕組み・設計・投資）」を必ず入れる
- ターゲットが実際に「嫌だな」と感じる具体的な場面（匂いが服につく・家族に指摘されるなど）を先に言語化してから逆転させると共感が生まれる
- 「真逆のポジション」を取れている場合、それ自体が強力な差別化になる
