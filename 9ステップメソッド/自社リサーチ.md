# 自社リサーチ ノウハウ

ジョブリスティング実践講座ステップ3「自社リサーチの課題」の進め方をベースに、
フィードバック・事例・AIプロンプト型で肉付けしたもの。

最終更新：2026年4月5日

> **このファイルの使い方**
> 読んでわかった気になっても、一人で完結させることはできない設計になっている。
> 「事前に何をすればいいかを把握する」「フィードバック後に振り返って整理する」の2つに使う。
> 各ステップで必ずフィードバックを受けながら進めること。

---

## 目的

- 求人でよく見かける情報（特徴）だけでなく、「ここで働きたい」と思ってもらえる情報を見つけること
- 多角的な視点で、ターゲットが疑問や不安を一切感じないように全ての情報を集める
- 他社にはなかなか真似できない独自の強みを見つけ出すことがゴール

最終目的は、**「なぜこの仕事か」「なぜこの会社か」を求職者が自分で判断できる根拠集を作ること**。  
応募理由・比較理由・信用理由の3つがそろった状態が完成形。会社説明や制度紹介の要約になっていたら失敗。

**おおよその所要時間（感覚値）：**
- 求人案件を持っている場合：未経験者1ヶ月程度・経験者2週間程度
- Zoomでヒアリングする場合：未経験者4回程度・経験者2回程度（1回1〜2時間）

---

## 必ず持つべき2つの基準

### 基準1：ターゲットの立場に立って情報を集める

広告は全て相手ベース。常にターゲットを頭に思い浮かべながら情報を出す。  
「もし昔の自分がターゲットと同じ立場だったら？」と置き換えるのも有効。

- あの時の自分だったら疑問を感じるところは？
- 不安に思っていたことは？
- 何が響く？逆に何が響かない？

**往々にしてターゲットのことを忘れてしまうので注意：**
- 職場内の人と話す感覚で情報を出してしまう
- 専門用語を使ったり、既に知っていることが前提で話すケースが多い
- 求職者は何も知らないので、「何も知らない人に伝わるように」情報を出す必要がある

### 基準2：全体の流れとカテゴリーを常に意識する

よくある問題：
- カテゴリーの中に違う情報が混ざっている
- 順番がおかしい（上から読んで理解できない）
- 情報が入り過ぎている（何が重要かわからない・重複・不必要な情報が多い）

## 進め方

### ステップ1：サンプルのマインドマップを参考に方向性とゴールのイメージを掴む

- 幹を見てから枝葉をざっと確認する（ゴールの確認）
- 幹のみにして全体の構造を確認し、自社に当てはめて集めるべき情報の構造を決める（方向性の確認）
- ゴールと方向性を確認したら、フィードバックを受ける

### ステップ2：カテゴリー毎に枝葉の情報を詰めていく

- 方向性が定まったら、カテゴリー毎に情報収集する
- 順番が関係しているので、上から順番に進める
- AIと壁打ちして一つのカテゴリーがまとまったら、フィードバックを受ける

**情報収集の4つのポイント（よくある不足箇所）：**

**① 難しい表現はNG。小学6年生でも理解できるように**
- 専門用語・会社独自の言葉はNG（経験者がターゲットでも、マイナーな専門用語は使わない方が良い）
- ネットを見ている時は注意力散漫。普段使わない難しい言葉があると途端に思考停止する
```
（AI活用）この内容を専門用語を使わず、小学6年生でも伝わるように表現してください
（AI活用）◯◯（ターゲット）が注意力散漫な状態でもすぐに理解できるように調整してください
```

**② 抽象的な情報を具体的にする**
- 「一人前」「高収入」「働きやすい」「人間関係が良い」→ 何が良いのかよくわからない
- 具体化の最終形は数値化。ターゲットが頭の中で画像としてイメージできるところまで
- 過去の賞与実績・定着率のデータ・口コミデータなど数値化できるところは必ず数値化する
```
（AI活用）この内容で具体性が足りないポイントを教えてください
```

**③ 必ず比較の情報を出す**
- 何かを伝える時は比較対象の情報があるとわかりやすくなる
- 逆にないと、良し悪しがわからないケースも多い
```
（他業種との比較）
◯◯（ターゲット）は一般的にどんな業種を比較検討しているのか？
◯◯（業種）を選ぶ人はどのような基準で選んでいますか？

（同業他社との比較）
◯◯（業種）の仕事をしたい人が応募先を探す基準を教えてください
◯◯（特徴）のある会社は一般的にどうしていますか？
```

**④ 理由の情報も出す**
- 業種選択・会社選択の際に「この理由がなければ納得してもらえない」ので選んでもらえない
- 特徴があるということは何かしらの理由があるはず。なんとなく付けている特徴でも全て出していく
```
◯◯（特徴・制度）を導入している会社は一般的にどんな理由で導入していますか？
例：家族手当を導入している会社は一般的にどんな理由で導入していますか？
```

**⑤ 特徴にはベネフィットと事例を出す**

ベネフィット＝その特徴があることでスタッフがどうなったか・どう変わったか（結果・変化）。  
ビフォー・アフターで出すとやりやすい。
```
前職：◯◯◯◯（問題のある状態）
現職：◯◯◯◯（改善された状態）
→ 一般的にこのスタッフはどのように変化していきますか？事例があれば教えてください。
```

**特徴→求職者視点への変換例（よくある会社側の情報の言い換え）：**

| 会社側の表現 | 求職者視点への変換 |
|---|---|
| 多店舗展開中 | 転勤なし、通いやすい店舗を選べる |
| 残業削減 | 毎日決まった時間に帰れるから、子育てや趣味と両立できる |
| 創業30年 | 経営基盤が安定しているので、安心して長く働ける |
| 表彰実績あり | スタッフの頑張りをちゃんと評価してくれる会社 |

**変換できない情報は省く：**
- 細かな業務内容の羅列（1日の掃除は○分、タオルは○回洗濯します）
- 求職者に無関係な経営理念（地域に愛される○○を目指しています）
- 社内向けの用語・スローガン

### ステップ3：情報整理しながら、足りない部分を埋めていく

**① ターゲットから見て情報が不足していないか？**
```
（AI活用）この内容で◯◯（ターゲット）から見て疑問や不安は感じないか？
```
- 疑問は感じないか？
- 不安は感じないか？
- 信頼できるか？

**② 競合他社と明確に差別化を図れる情報は揃っているか？**
```
（AI活用）以下の内容で競合他社と明確に差別化を図れるポイントはどこだと思いますか？不足しているところはありますか？
```

**③ ビッグプロミス（候補）を出せているか？**
```
（AI活用）◯◯（ターゲット）にとって当社でしか得られない強みを一言で表すとなんといえますか？以下の内容をみて5つ出してください。
```

**④ 情報整理の手順：**

情報は集まったのに「何を訴求すべきかわからない」という状態になりやすい。これは整理を飛ばして仮説を立てようとしているため。**削除→グループ化→並べ替え→要約**の順を守ることで仮説の質が上がる。

```
1. 声に出して上から順番に読む
2. 不必要な情報を削除（ターゲットに必要な情報のみ・重複は削除）
3. グループ化（共通点・パターンを見つける。別カテゴリーへ移動または新カテゴリーを作る）
4. 流れを意識して並べ替える（時系列順 / 優先順位順 / 因果関係順）
5. 情報を「一言でまとめる」
```

「整理する」ことが目的ではなく「AIに渡す仮説が良くなる」ためにやる。  
わからなくなったらAIに「この情報を整理するとどうなりますか？」と投げてヒントにする。ただし、AIはそこまで整理してくれないので、自分の頭で理解することが前提。

→ 詳細は `仮説思考.md`「ステップ2：観察」を参照

**良いと思うアイデアが出たらフィードバックを受ける。**

## 次のステップに進むためのチェックリスト

- ✅ 求職者目線で「この会社で働きたい！」と思われるネタが集まっているか？
- ✅ 求職者からみて疑問や不安を感じる要素はないか？
- ✅ 小学6年生が見てもわかりやすく情報整理されているか？
- ✅ 競合と比べて明らかに強いと言える武器は見つかっているか？
- ✅ ビッグプロミスの候補を出せているか？

## 補足・深掘り

### 自社リサーチの9つのチェックリスト（深掘りチェック）

各特徴に対して確認する：

**深掘りチェック（3項目）：**
- [ ] 理由は入っているか？（他業種との比較・競合他社との比較・過去の自社の問題・求職者が抱えている不安・世間のイメージ など）
- [ ] ターゲットの立場で書かれているか？（特徴を出しただけで終わらず、ターゲットにとって何が良いかに変換する）
- [ ] ベネフィット（結果・変化）は根源的欲求に届いているか？（浅くならないように）
```
（AI活用）この（特徴）があることで（ターゲット）にとってどんな変化・結果が得られますか？根源的欲求に届くまで深掘ってください。
```

**表現チェック（3項目）：**
- [ ] 抽象的な表現になっていないか？
- [ ] 専門用語を使っていないか？
- [ ] 小学5年生でもわかる内容になっているか？

**整理チェック（3項目）：**
- [ ] 同じカテゴリーの中に異なる要素が混ざっていないか？
- [ ] 重複している情報を一つにまとめているか？
- [ ] ロジカル（因果関係や時系列）に繋がっているか？（枝だけでなく幹もチェック）

**全9項目中7項目以上Yesで提出可。**

### 繰り返し出るNGパターン

- 「○○があります」だけで「なぜ」がない
- 比較なしに自社の特徴だけ並べる
- ベネフィットを書かない（「だから求職者にとってどう良いか」まで落とさない）
- 数値化できるところを抽象語で止める
- スタッフインタビューをせずに会社側の思い込みで書く

### 弱い状態（よくある不十分パターン）

- ターゲット設定はあるが、求職者理解が浅い（願望だけで悩み・不安を見ていない）
- 特徴が募集要項レベルに偏っている（業種特性・制度・研修・評価・人物要素まで拾えていない）
- ベネフィットが浅く、求職者が自分ごと化できない
- ビッグプロミスが見えず、全体の訴求軸がぼやけている
- 数値・流れ・事例・条件の具体性が不足していて、働く姿が見えない

### ズレている状態（方向性が間違っているパターン）

- 自社リサーチが会社説明や制度紹介の要約になっている
- 求職者視点ではなく、会社が言いたいこと中心で整理している
- 業種の魅力と会社の魅力を混同している
- 給与・休日など一部の欲求だけで求職者理解を済ませている

### 特徴・ベネフィット・ビッグプロミス・独自メソッドの関係

| 概念 | 定義 |
|---|---|
| **特徴（Feature）** | 事実の情報。勤務地・仕事内容・制度など。これだけでは選ばれない |
| **ベネフィット（Benefit）** | 特徴があることで、求職者の生活や状況がどう変わるか。欲求・痛みに届くまで掘り下げる |
| **ビッグプロミス** | 複数のベネフィットを統合した「一言の大きな約束」。タイトルやファーストビューに使う |
| **独自メソッド** | ビッグプロミスを実現する「自社ならではの仕組み・方法・プロセス」。ベネフィット（結果）と混同しない |

**未経験者採用では整理の順序：「なぜこの仕事か」→「なぜこの会社か」の順で整理する。**  
この順序が崩れると、業種への納得がないまま会社比較に入り、訴求が届きにくくなる。

### 証拠の種類と集め方

| 種類 | 例 |
|---|---|
| **目に見える証拠** | スタッフのインタビュー動画・「入社から3年の平均在籍率○%」・写真 |
| **概念的証拠** | 「なぜこの制度があるのか」の経緯・ストーリー・理由 |

**約束が大きいほど、証拠も大きくしなければならない。**
